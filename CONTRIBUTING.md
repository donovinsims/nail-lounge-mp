# Contributing

> Last updated: 2026-07-07 (3:30 PM CST)

## Before you start

- Read [`AGENTS.md`](./AGENTS.md) and [`docs/ARCHITECTURE.md`](./docs/ARCHITECTURE.md).
- Use **Bun** for everything locally (`bun install`, `bun run ...`). Never npm or yarn for local dev.

## Verification checklist (every change)

All four must exit clean before pushing — CI enforces the same gates:

```bash
bun run lint        # eslint . --max-warnings 0 (zero errors AND zero warnings)
bun run typecheck   # tsc --noEmit
bun run test        # vitest run (82+ tests)
bun run build       # production build
```

Hooks: pre-commit runs lint-staged; pre-push runs the secret scan (`scripts/check-secrets.sh`), build, and typecheck.

## Code standards

- **No `any`.** `@typescript-eslint/no-explicit-any` is an **error**. Use the generated `Database` types (`src/integrations/supabase/types.ts`), a named row type, or `unknown` + narrowing.
- **Zod schemas live in source and are exported.** Tests import them from `booking.functions.ts` / `admin-crud.functions.ts` — never replicate a schema inline in a test.
- **Error handling:** `catch (err: unknown)` + `getErrorMessage(err, fallback)` from `src/lib/error-handler.ts`. Never `catch (e: any)`.
- **Async:** `async/await` over `.then()` chains (the lazy server-entry import in `src/server.ts` is the known exception).
- **Formatting:** Prettier — 100 print width, double quotes, trailing commas. **LF line endings only** (a single CRLF fails CI via the Prettier lint rule).
- **Route files** (`src/routes/`) must not export non-component values — shared constants/hooks/variants live in companion files (see `src/components/ui/*.variants.ts` and `*.context.ts`).

## Dual-lockfile rule (important)

CI installs with **Bun** (`bun.lock`); **Vercel builds with npm** (`package-lock.json` takes install priority). After ANY dependency change, update **both** lockfiles and commit them together:

```bash
bun install
npm install --package-lock-only
```

If they drift, CI and Production build different dependency trees.

## Never hand-edit

- `src/routeTree.gen.ts` — generated by TanStack Router on build
- `src/integrations/supabase/types.ts` — generated by the Supabase CLI (`supabase gen types`)
- The `prepare` script in `package.json` — it must stay graceful for Vercel (no `.git`/husky present there)

## Product compliance

- **Never gate review requests on the rating.** Every client gets the same review invitation regardless of score (Google review policy + FTC 16 CFR Part 465). Low ratings additionally alert the owner — that is the recovery path, not suppression.
- No fabricated proof, testimonials, or scarcity anywhere in copy.

## Pull request checklist

- [ ] `bun run lint` — 0 errors, 0 warnings
- [ ] `bun run typecheck` — clean
- [ ] `bun run test` — all passing (new logic has tests)
- [ ] `bun run build` — succeeds
- [ ] No new `any`, no `eslint-disable` as a fix
- [ ] Dependency changes: both lockfiles updated
